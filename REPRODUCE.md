# How to reproduce

**Shortcut — download the pre-built manifests instead of regenerating:**
```bash
cd code && aws s3 sync --no-sign-request s3://nopert/manifests/ .
```
then jump straight to the verifiers (the `python verify_*.py` / `assembly_verify.py` steps below).
The rest of this file documents regenerating each manifest from source.

All commands are run from `code/` unless noted. Manifests are written next to the scripts (e.g.
`code/recover_out/`), matching the paths the verifiers expect. Nothing here needs network access or
hosted data — every manifest is regenerated by its driver. Wall-clock figures are for a 10-core
2023-class laptop; the pure-Python path works everywhere, the optional Cython build (below) is ~5×.

```bash
pip install -r ../requirements.txt
# optional ~5x speedup for the interval kernels:
python build_ext.py           # builds fast_interval_cy, tm2_cy, g_aa_cy, g_da_cy in place
```

Each step prints a summary you can check against the **expected counts** in bold.

## IN-3 — far manifest (det-dual) + low-δ recovery

The base annulus sweep is the Rust prover; the low-δ recovery and its verification are Python.

```bash
# (a) build + run the DK det-dual sweep over the wall band δ∈[6e-3,3e-2] × (φ_w±8e-3):
cd ../rust/dk_prover && cargo build --release
#   dkrun writes the box manifests consumed below (see src/bin/dkrun.rs for the CLI);
#   this reproduces certificates/dk_full/boxes/*.jsonl.gz.
cd ../../code

# (b) extract the sweep's 'stuck' cells and recover them (γ-refine + adaptive + LP):
python -c "import json,gzip,glob; \
open('dk_full/stuck_all.jsonl','w').writelines( \
 json.dumps([r['cen'],r['hw']])+'\n' for f in sorted(glob.glob('dk_full/boxes/box_*.jsonl.gz')) \
 for r in map(json.loads, gzip.open(f,'rt')) if r.get('kind')=='stuck')"
for s in $(seq 0 8); do python recover_lowdelta.py $s 9 recover_out & done; wait
python recover_fix.py               # re-close the deep φ_w-core residuals → rec_fix.jsonl

# (c) independent replay verification of the whole recovery layer:
python verify_recover.py --outdir recover_out --sample 2000
```
**Expected:** the Rust sweep leaves **809,815** stuck cells; **every one has margin_lp > 0**
($\ge 1.96\text{e-}7$ — the obstruction is enclosure resolution, not the mathematics). The recovery
closes all of them → **1,038,991** certified leaves, **207** deep-core residuals re-closed by
`recover_fix`, **0** unclosed. `verify_recover` → V1 coverage **809,815 = 809,815**, V2 dyadic
tiling 0 bad, V3 0 unclosed fails, V4 `certify_cell` re-exec 0 bad → **VERDICT PASS**.
(Recovery is ~3 h on 9 cores in pure Python; the Cython build cuts this substantially.)

The high-δ det-dual region is checked separately with `dk_verify.py MANIFEST.jsonl` (tiling + stuck=0
+ per-cell det-dual re-check + corner-skip re-check → `VERDICT PASS`).

## The (SB-box) delta-continuum + analytic constant C4

The (SB-box) extension (paper §15.6) is certified over the full **delta-continuum** by a direct
interval-delta sweep (the same two-symbol Taylor-form / WS-2 dual-weight kernel as the base sweep;
the residual is linear in the flags, so it stays tight over delta-intervals). The one analytic
constant of the local theorem, the order-4 Taylor-tail C4, is proved by its own exact-interval
certificate (paper §13.4):

```bash
python ivd_extension_sweep.py   # SB-box delta-continuum [1.28e-3, 5e-2] x [0,pi]: 10,560 cells, 0 fails
python c4_taylor.py             # C4 <= 53 < 60      (order-4 tail; per-config ~52.99, ~mins)
```
**Expected:** `ivd_ext_out/SUMMARY.json` records `fails: 0`, `min_slack ~2.6e-6`, `max_eta ~9.9e-4`
(`verdict: CERTIFIED interval-delta: 0 FAILS over the continuum`); `c4_taylor_out.json` records
`C4_bound <= 53` (`verdict: PROVED C4 <= 52.998 < 60`). Both are consumed by `assembly_verify.py`.
(The earlier `adelta_run_all.py` flag-acceleration bound `A_delta <= 8`, and the 30 point-delta
`sbbox_ext/` manifests, remain in the bundle as an independent cross-check of the continuum sweep;
they are no longer load-bearing.)

```
ASSEMBLY VERDICT: COMPLETE (all inputs verified, interfaces gap-free)
```

`assembly_verify.py` reads each input's verifier output (the `reports/` JSONs and the per-manifest
`.done` stats) and checks the two-radius straddle interface inequalities [S1]–[S4]. It is the script
whose output *is* the paper's computational claim.
